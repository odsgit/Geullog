# Geullog 개발 기록

버전별 개발 내용을 정리한 문서입니다. 새 버전 작업이 진행될 때마다 이 파일에 이어서 기록합니다.

## v2.0 (진행 중)

- **생성 폼 간소화 (2026-07-05)**: 프로덕션 실사용 피드백("필수 입력 단계가 너무 많다") 반영.
  필수 입력을 주제/글종류/타겟독자 3개로 줄이고, 톤/스타일/분량/언어/전개방식/작가스타일/
  문체프리셋/이미지는 전부 기본 접힘 상태인 "고급 설정" 아코디언으로 이동(전부 선택 사항).
  "AI에게 추가 요청"(자유 입력), SEO 키워드, 출력 형식(마크다운/일반 텍스트) 필드 신규 추가.
  `record_generation` RPC 파라미터를 필수/옵션 순으로 재정렬(스타일/톤/분량/언어/전개방식
  전부 `default null`로 변경). **참고 URL(참고 자료) 필드는 의도적으로 제외** — 서버가 사용자
  입력 URL을 직접 fetch하면 SSRF 위험이 있어 도메인 화이트리스트/타임아웃 등을 갖춘 전용
  구현이 필요하므로 별도 작업으로 남김. **검증**: 실제 브라우저로 최소 3필드만 입력한 생성이
  정상 동작함을 확인, "반말로 작성해줘" 추가 요청이 실제로 반말체 출력을 만들어내는 것과
  `plain` 출력 형식에서 마크다운 기호가 전혀 없는 것까지 확인.
- **크레딧 충전 기능 (2026-07-05)**: `/settings` 페이지 신규 추가. 결제 연동 전 데모 단계로,
  10 / 30 / 100 / 300 크레딧 패키지 버튼을 누르면 즉시 충전됨. `charge_credits` RPC가 허용된
  금액만 서버에서 화이트리스트 검증(클라이언트 우회 호출 방어), `usage_logs`에 충전 내역 기록.
  헤더의 크레딧 배지에서 바로 진입 가능.
- **v2.0 리팩토링 통합본 (2026-07-05, `Geullog_v2.0_지시문.md` STEP A~F, H)**: v1.0 운영 중
  발견된 6가지 문제 + 버전 되돌리기 기능.
  - **필드 중복 정리**: 서술 유형(narrative_type: 서사/묘사/설명/논증)과 세부 장르를 DB·타입·UI
    전부에서 완전 제거, 스타일 옵션의 정보전달형/스토리텔링형도 제거(narrative_type과 개념 중복).
    대신 단일 "전개 방식"(development_structure) 10종(3단구성/5단구성/두괄식/역피라미드/PREP/
    기승전결/소설5단/영웅의여정/병렬식/시공간흐름)으로 통합.
  - **작가 스타일 가드레일**: doc_type을 창작형(blog)/실용형(product·sns·email·resume·press)으로
    분류, 실용형은 작가 스타일 대신 문체 프리셋(간결체/서정체/유머러스체/신뢰감 있는 문체) 노출.
  - **작가 스타일 티어링**: author_styles에 tier(tier1/tier2)+traits 컬럼 추가. 데이터가 희소한
    5명(크라스나호르카이·채만식·최명희·바누 무쉬타크·양솽쯔)은 tier2로 표시, 구체적 문체
    traits를 프롬프트에 불릿으로 직접 주입(이름만 언급하는 대신). 나머지는 tier1 기본값.
    실제 검증: 크라스나호르카이 선택 시 생성물이 실제로 마침표 없는 극단적 만연체로 나옴.
  - **이미지 업로드 모드 분리**: OCR(사진 속 글자 읽기)/분위기 묘사 라디오 버튼 추가, 미선택 시
    제출 버튼 비활성화. Vision 프롬프트를 모드별로 분기. 실제 검증: 텍스트 이미지+OCR 모드 시
    이미지 속 문구가 결과물에 그대로 반영됨 확인.
  - **마크다운 구조 강제**: 전개방식별 프롬프트가 실용형은 소제목(##) few-shot, 문학형은 빈 줄
    문단 구분만 지시하도록 동적 삽입. `react-markdown`(스트리밍 미리보기)과 `tiptap-markdown`
    (에디터, 마크다운을 실제 heading/paragraph 노드로 파싱)으로 프론트 렌더링 교체 —
    `/trial`·데모 히어로도 동일하게 마크다운 렌더링으로 교체(통글 텍스트 문제 해결).
  - **버전 되돌리기**: `generations.current_version_id` + `generation_versions.version_type`
    ('generated'|'reverted'|'tone_adjusted') 추가. 되돌리기는 기존 버전을 절대 수정/삭제하지
    않고 새 row만 추가(append-only), `revert_generation_version` RPC(크레딧 미차감). 히스토리
    상세에 버전 타임라인 UI. 실제 검증: A→B(재생성)→C(톤조정)→A로 되돌리기 시 4개 버전 모두
    보존 + 현재 버전 포인터가 되돌린 버전(A')을 정확히 가리킴, 다른 계정으로 되돌리기 RPC 직접
    호출 시 `not_found`로 차단됨(RLS 우회 불가) 확인.
  - **검증**: Playwright(실제 브라우저) + `wrangler pages dev`(Cloudflare Pages Functions
    로컬 실행, `/api/generate`는 순수 vite dev로는 동작 안 함) + Supabase Admin API로 테스트
    유저 2명 생성해 교차 계정 RLS까지 실제 확인, 종료 후 전부 정리. 검증 중 재생성/톤조정 후
    히스토리 페이지의 버전 타임라인이 갱신되지 않는 실제 버그를 발견해 즉시 수정
    (`GenerationResult`에 `onVersionCreated` 콜백 추가).

## v1.0 (2026-07-05 태그)

AI 글쓰기 생성 SaaS의 첫 정식 버전. 핵심 생성 기능부터 성장 루프 기능까지 전체 완료.

**기반 (#1~#7)**
- 프로젝트 스캐폴딩, GitHub Actions CI, Cloudflare Pages 배포(Git integration 자동 배포)
- Supabase 스키마/마이그레이션/타입 세이프 클라이언트, GA4 + Microsoft Clarity 연동

**인증 (#6)**
- 이메일/비밀번호 로그인 및 회원가입, Google OAuth, 보호된 라우트, 가입 시 프로필 자동 생성(크레딧 10 기본 지급)

**핵심 생성 플로우 (#8~#12)**
- 주제/글종류/스타일/톤/타겟독자/분량/언어를 입력하는 생성 폼(react-hook-form + zod)
- OpenAI 연동 SSE 스트리밍 생성, 크레딧 원자적 차감(동시 요청 레이스 컨디션 방어 검증 완료)
- 이미지 업로드 + GPT-4o Vision으로 사진 내용을 반영한 글 생성
- 리치 텍스트 에디터(Tiptap), 재생성/톤 조정, .txt·.docx 내보내기, 생성 히스토리

**성장 루프 (#13~#18)**
- 바이럴 공유 페이지(OG 메타 태그, 비공개 전환 가능)
- 리퍼럴 시스템(추천 링크로 가입 시 추천인/피추천인 모두 +5 크레딧)
- 비로그인 무료 체험(IP 해시 기반 1회 제한)
- 공개 템플릿 갤러리
- SEO용 MDX 블로그 콘텐츠 허브
- 주간 사용량 요약 이메일(pg_cron + Resend, Vault 기반 시크릿 인증)

**UI 리뉴얼 (#19~#20, #22)**
- 1차 Neubrutalism → 실사용 후 폐기 → 2차 웜톤 에디토리얼 디자인으로 확정
- `/login`에 로그인 없이 체험 가능한 인터랙티브 데모 히어로 추가

**글쓰기 축 확장 (#23~#30)**
- 마크다운 기호 출력 방지
- 작가 스타일 선택(실제 문체 데이터 50인 기반)
- 서술 유형(서사/묘사/설명/논증) 및 세부 장르 선택
- 장문 이어쓰기(시리즈) 기능, 이전 파트 자유 열람

## 작성 규칙

- 새 버전 작업을 시작하면 이 파일 상단에 `## v{N}.{M} (진행 중)` 섹션을 추가하고, 완료되는
  기능마다 한두 줄로 이어서 기록한다.
- 버전이 마무리되어 태그를 찍으면 섹션 제목의 "(진행 중)"을 "(YYYY-MM-DD 태그)"로 바꾼다.
- 각 항목은 "무엇을 왜"보다 "무엇이 실제로 동작하는지" 위주로 간결하게 쓴다(자세한 구현
  맥락은 커밋 메시지/PR에 있음).
